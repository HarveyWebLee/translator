## 3. 启动流程

### 3.1 拉取制品

> TODO：明确制品形态（Docker 镜像 / pnpm tarball / 直接 git pull）。

```bash
# 方案 A：Docker 镜像（推荐）
docker pull registry.example.com/translator/server:<version>

# 方案 B：源码 + pnpm
git clone <repo>
cd translator
pnpm install --frozen-lockfile --prod
pnpm --filter @translator/server build
```

### 3.2 执行数据库迁移

```bash
# 显式 deploy（不会 prompt），用于生产
pnpm --filter @translator/server prisma:deploy
```

> 注意：首次部署时校对 baseline；若历史数据库非由本项目创建，需用 `prisma migrate resolve` 标定基线。

### 3.3 启动应用

```bash
# Docker
docker run -d --name translator-server \
  -p 19696:19696 \
  --env-file /etc/translator/.env \
  --restart=always \
  registry.example.com/translator/server:<version>

# pm2（裸机）
pm2 start apps/server/dist/main.js \
  --name translator-server \
  --env production \
  -i max
```

### 3.4 健康检查

```bash
curl -fsS https://<api-host>/api/v1/health
# 期望：{ "ok": true, "data": { "status": "up", "ts": ... } }
```

`/api/docs`（Swagger）建议在生产关闭或仅对内网开放。

## 4. 反向代理示例（Nginx）

```nginx
# TODO: 根据实际域名 / TLS 证书路径补全
upstream translator_api {
  server 10.0.0.11:19696;
  server 10.0.0.12:19696;
  keepalive 64;
}

server {
  listen 443 ssl http2;
  server_name api.translator.example.com;

  ssl_certificate     /etc/ssl/translator/fullchain.pem;
  ssl_certificate_key /etc/ssl/translator/privkey.pem;

  # SSE 关键：禁用代理缓冲，提高超时
  proxy_buffering off;
  proxy_read_timeout 600s;
  proxy_send_timeout 600s;
  proxy_http_version 1.1;

  location / {
    proxy_pass http://translator_api;
    proxy_set_header Host              $host;
    proxy_set_header X-Real-IP         $remote_addr;
    proxy_set_header X-Forwarded-For   $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }
}
```

## 8. 故障速查

> TODO：每条加入实际命令与 Runbook。

| 现象                               | 优先排查                                                       |
| ---------------------------------- | -------------------------------------------------------------- |
| 401 飙升                           | JWT 密钥配置、时钟漂移、refresh token 表损坏                   |
| 翻译 SSE 大面积失败                | LLM 上游限流；Nginx `proxy_buffering` 是否被改回 on            |
| `/api/v1/translation/stream` 卡死  | service-worker 没有 `req.on('close')` 释放（已修复，回归校验） |
| Prisma `connection pool exhausted` | 调大 `connection_limit`；检查长事务                            |
| 内存持续上涨                       | SSE 会话未释放；监听 `subscribed` Map 大小                     |
| 扩展加载后无反应                   | 检查 manifest content_scripts 路径；F12 看 service worker      |

## 9. 安全基线

- [ ] 所有外网入口走 TLS 1.2+

- [ ] LLM Key 仅以环境变量注入；不写入日志
