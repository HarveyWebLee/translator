# AI 翻译助手 — 需求功能文档

> 文档版本：v2.0.0  
> 产品形态：浏览器扩展（Chrome / Edge，Manifest V3）+ 远程公共后端（NestJS）  
> 目标用户：需要阅读英文网页、希望获得高质量中文译文的中文母语用户

---

## 1. 产品概述

### 1.1 产品定位

**AI 翻译助手** 是一款由浏览器扩展前端与公共云端后端共同组成的产品。当用户浏览英文网站时，扩展可自动识别页面语言，在用户确认后由后端调用对应翻译引擎（免费 SDK 或 LLM），将页面正文**流式逐步**翻译为简体中文，并提供**划词词典式翻译**。

### 1.2 核心价值

| 价值点         | 说明                                                                                       |
| -------------- | ------------------------------------------------------------------------------------------ |
| 智能识别       | 无需用户手动指定语言，结合 `html[lang]` 与正文采样判断是否为英文页                         |
| 用户可控       | 翻译前需用户确认，避免误译与不必要的 API 消耗                                              |
| 多引擎选择     | 一套抽象层接入 DeepSeek / OpenAI / Anthropic / Ollama / Google / LibreTranslate            |
| 会员分级       | 免费 / 初级 / 高级三档，对应不同的引擎与 API Key 来源，控制成本与商业模式                  |
| 按需流式翻译   | 后端通过 SSE 推送，前端视口优先翻译、滚动续译，首屏可读时间最短                            |
| 学习辅助       | 划词模式提供音标、词性、例句与上下文释义                                                   |
| 跨设备账户     | 邮箱密码注册（OAuth / 短信预留），偏好设置随账户绑定                                       |

### 1.3 非目标（当前版本不做）

- 不接入支付渠道，订阅升级仅留接口骨架
- 不支持离线翻译（除本地 Ollama 模式）
- 不保证 iframe、Shadow DOM、Canvas 内文本的完整覆盖
- 不提供持久化「双语对照」存档（刷新页面即恢复原文）
- 不向用户开放 Prompt 模板编辑（仅平台内置）

## 2. 用户角色与会员体系

### 2.1 用户角色

- **匿名访客**：未登录时仅展示落地页与登录引导，无法发起任何翻译请求
- **免费会员**：注册即拥有，可使用 Google 公开端点 / LibreTranslate 翻译
- **初级会员**：付费档位（暂不接入支付）；可使用通用 LLM（DeepSeek/OpenAI/Anthropic/Ollama 等），**API Key 由用户自带**，保存在扩展本地
- **高级会员**：付费档位；可使用全部模型，**高级模型 Key 由开发者后端持有**，用户无需配置
- **管理员**：仅运维内部，负责模型目录维护、配额调整、用户审计（管理后台暂未实现）

### 2.2 会员能力对照

| 能力                   | 免费 | 初级           | 高级             |
| ---------------------- | ---- | -------------- | ---------------- |
| Google 公开端点翻译    | ✓    | ✓              | ✓                |
| LibreTranslate         | ✓    | ✓              | ✓                |
| DeepSeek/OpenAI/Anthropic 等 LLM | ✗ | ✓（自带 Key） | ✓（系统 Key）    |
| Ollama 本地模型        | ✗    | ✓（自带服务）  | ✓                |
| 划词翻译（LLM 解析）   | 受限 | ✓              | ✓                |
| 日翻译额度             | 低   | 中             | 高               |

### 2.3 注册与登录

| 方式              | 当前状态                                      |
| ----------------- | --------------------------------------------- |
| 邮箱 + 密码       | 已实现：注册 / 登录 / 刷新 Token / 注销       |
| Google OAuth      | 骨架：路由与策略已搭，待接入官方 SDK 与回调   |
| GitHub OAuth      | 骨架：同上                                    |
| 手机短信验证码    | 骨架：服务接口已就位，待接入短信通道（阿里云/腾讯云） |

### 2.4 典型使用场景

| 场景 ID | 描述                                                | 期望结果                                                     |
| ------- | --------------------------------------------------- | ------------------------------------------------------------ |
| S1      | 用户首次安装扩展并打开选项页                        | 引导注册或登录；未登录无法翻译                               |
| S2      | 免费用户打开英文页面                                | 自动检测 → 顶部提示 → 调用免费引擎流式翻译                   |
| S3      | 初级用户切换到 DeepSeek 模型并填入 API Key          | Key 保存在 `chrome.storage.local`；翻译请求时由扩展上送      |
| S4      | 高级用户切换到 `claude-3-opus` 等高级模型           | 后端用 env 中的 Key 调用，无需用户配置                       |
| S5      | 用户选中页面英文单词                                | 弹出词典面板（受会员等级影响：免费仅基础翻译，付费有详解）   |
| S6      | 阅读含代码块的文档                                  | `<pre>` / `<code>` 内代码保持英文原文，不被翻译              |
| S7      | 用户在多设备登录同一账户                            | 偏好（启用、自动提示、当前模型）同步                         |
| S8      | 用户长时间未活动                                    | Access Token 过期 → 静默 refresh → 失败时跳转登录            |

## 3. 功能需求清单

### 3.1 账号与会员（F-AUTH）

| 编号        | 需求                                              | 优先级 | 验收标准                                                  |
| ----------- | ------------------------------------------------- | ------ | --------------------------------------------------------- |
| F-AUTH-01   | 邮箱密码注册                                      | P0     | argon2 加密、密码强度校验、邮箱唯一性约束                 |
| F-AUTH-02   | 邮箱密码登录 + JWT                                | P0     | access token + refresh token rotation                     |
| F-AUTH-03   | 选项页展示当前用户及会员等级                      | P0     | 头像、邮箱、tier、过期时间                                |
| F-AUTH-04   | 注销并清空本地 token                              | P0     | 后端撤销 refresh token，前端清空 storage                  |
| F-AUTH-05   | OAuth / SMS 登录入口（按钮可见、点击提示开发中）  | P2     | UI 占位，后端策略骨架就位                                 |
| F-AUTH-06   | 订阅升级页面（仅展示套餐与「即将上线」）          | P2     | BillingModule 接口可调，不真正扣费                        |

### 3.2 语言检测（F-DETECT）

| 编号        | 需求                                                    | 优先级 | 验收标准                                               |
| ----------- | ------------------------------------------------------- | ------ | ------------------------------------------------------ |
| F-DETECT-01 | 读取 `html[lang]`、`meta content-language`、`og:locale` | P0     | 声明为 zh/ja/ko 时判定为非英文；声明为 en 时判定为英文 |
| F-DETECT-02 | 对可见正文采样（排除 script/style/pre/code 等）         | P0     | 采样长度上限约 8000 字符                               |
| F-DETECT-03 | 基于拉丁字母 / CJK 占比启发式判断                       | P0     | 拉丁占比 ≥55% 且中文占比 <12% 时判定为英文             |
| F-DETECT-04 | 正文过少时降低置信度，不强行提示                        | P1     | 有效字符 <40 或总采样 <80 时不自动弹条                 |
| F-DETECT-05 | 页面加载后延迟检测（800ms / 2500ms 两次）               | P1     | 适配 SPA 延迟渲染                                      |
| F-DETECT-06 | Popup 内展示当前标签页语言状态                          | P2     | 显示疑似英文/非英文及 lang 属性                        |

### 3.3 整页翻译（F-PAGE）

| 编号      | 需求                                                    | 优先级 | 验收标准                                                          |
| --------- | ------------------------------------------------------- | ------ | ----------------------------------------------------------------- |
| F-PAGE-01 | 检测到英文后顶部显示提示横幅                            | P0     | 含「翻译为中文」「暂不翻译」按钮                                  |
| F-PAGE-02 | 用户确认后才开始翻译                                    | P0     | 未点击前不调用 API                                                |
| F-PAGE-03 | 收集含英文字母的文本节点                                | P0     | 单段长度 ≥2，且含 `[a-zA-Z]`                                      |
| F-PAGE-04 | 跳过不应翻译的区域                                      | P0     | `pre/code/script/style` 等及其**所有子节点**（含 hljs 嵌套 span） |
| F-PAGE-05 | 视口内 + 下方 20% 预加载区优先翻译                      | P0     | 使用 IntersectionObserver + 滚动防抖                              |
| F-PAGE-06 | 后端通过 **SSE** 流式回传，每个分段单独事件             | P0     | `segment-done`、`done`、`error` 事件名规范，前端可即收即渲染      |
| F-PAGE-07 | DOM 替换时保留原文                                      | P1     | `data-ds-original` 存原文，`data-ds-translated` 标记已译          |
| F-PAGE-08 | 支持 SPA / 动态内容                                     | P1     | MutationObserver 刷新待译注册表                                   |
| F-PAGE-09 | 用户可停止逐步翻译                                      | P2     | 点击「停止逐步翻译」后中止 SSE 流并不再发起新批次                 |
| F-PAGE-10 | 刷新页面恢复原文                                        | P1     | 不持久化修改，依赖浏览器刷新                                      |

### 3.4 划词翻译（F-SELECT）

| 编号        | 需求                                                | 优先级 | 验收标准                       |
| ----------- | --------------------------------------------------- | ------ | ------------------------------ |
| F-SELECT-01 | 选项页可开关划词翻译                                | P0     | 默认关闭，需用户主动开启       |
| F-SELECT-02 | 鼠标选中英文后延迟 180ms 弹出面板                   | P0     | 避免与双击选词冲突             |
| F-SELECT-03 | 返回词典式 JSON：音标、词性、释义、例句、上下文解释 | P0     | 面板分区展示词典区与 AI 解释区 |
| F-SELECT-04 | 面板内可切换模型并重新请求                          | P1     | 切换后按当前选区重译           |
| F-SELECT-05 | 支持固定面板、复制、TTS 朗读                        | P2     | 固定后滚动不关闭；Esc 关闭     |
| F-SELECT-06 | 选区过长时截断至 400 字符                           | P1     | 防止超出模型上下文             |

### 3.5 模型管理（F-MODEL）

| 编号       | 需求                                                         | 优先级 | 验收标准                                                              |
| ---------- | ------------------------------------------------------------ | ------ | --------------------------------------------------------------------- |
| F-MODEL-01 | 后端维护模型目录（`model.catalog.ts`），含 tier 与 keySource | P0     | 通过 `/api/v1/llm/models` 输出供前端渲染                               |
| F-MODEL-02 | 前端按用户 tier 过滤模型选项                                 | P0     | 免费用户仅看到免费引擎；超过自身等级的模型置灰并提示升级               |
| F-MODEL-03 | 初级用户为需要自带 Key 的模型录入 API Key                    | P0     | 仅本地存储（`chrome.storage.local`），不上传后端                       |
| F-MODEL-04 | 高级模型自动使用后端 env 中的 Key                            | P0     | 用户无感知，请求体不含 Key                                             |
| F-MODEL-05 | 切换模型对划词与整页翻译同时生效                             | P1     | 偏好同步至 `chrome.storage.sync`                                       |

### 3.6 配置与管理（F-CONFIG）

| 编号        | 需求                                  | 优先级 | 验收标准                       |
| ----------- | ------------------------------------- | ------ | ------------------------------ |
| F-CONFIG-01 | 选项页：账户 / 模型 / 行为三个 Tab    | P0     | Antd Tabs，可分别配置          |
| F-CONFIG-02 | 总开关：启用/禁用扩展                 | P0     | 禁用后请求拒绝                 |
| F-CONFIG-03 | 自动提示开关                          | P0     | 关闭后不自动弹横幅             |
| F-CONFIG-04 | Popup 快捷开关与「翻译当前页」        | P1     | 脚本未注入时自动 executeScript |
| F-CONFIG-05 | 后端服务地址可通过 `.env` 配置        | P1     | `VITE_API_BASE_URL`             |

### 3.7 安全与合规（F-SEC）

| 编号     | 需求                                                  | 优先级 | 验收标准                                       |
| -------- | ----------------------------------------------------- | ------ | ---------------------------------------------- |
| F-SEC-01 | 用户自带 API Key 仅保存本地（`chrome.storage.local`） | P0     | 不上传后端，不进入 sync 同步通道               |
| F-SEC-02 | 后端 .env / 数据库凭据由运维管理，不入仓              | P0     | 项目根 `.gitignore` 包含 `.env` 与 `*.local.*` |
| F-SEC-03 | 所有 LLM 请求通过受信任的后端代理或浏览器扩展直发     | P0     | 防止内容脚本与第三方 API 的 CORS / CSP 问题    |
| F-SEC-04 | 所有受保护接口需 JWT 校验                             | P0     | `JwtAuthGuard` 全局启用，`@Public()` 例外      |
| F-SEC-05 | SSE 接口支持 token via query（EventSource 兼容）      | P1     | 后端 `JwtStrategy` 支持 header / query 两种    |
| F-SEC-06 | 密码 argon2 哈希、敏感字段不在日志中输出              | P0     | LoggingInterceptor 黑名单字段                  |

## 4. 交互与界面需求

### 4.1 顶部提示横幅（Content Script）

- 固定于页面顶部，不遮挡主要导航（z-index 高于正文）
- 状态：待翻译 → 逐步翻译中（进度）→ 翻译完成 / 已停止
- 加载态：主按钮禁用并显示「正在翻译…」

### 4.2 弹出窗口（Popup，React + Antd）

- 顶部：当前页语言检测状态圆点（绿/灰）+ 简要说明
- 中部：开关（启用扩展、自动提示、划词翻译）
- 底部：按钮（翻译当前页 / 打开选项页）
- 未登录时显示「请先登录」并跳转选项页

### 4.3 选项页（Options，React + Antd）

- Tab 1 **账户**：登录 / 注册表单、当前用户信息、注销
- Tab 2 **模型**：按 tier 分组的 Provider 卡片、当前选中模型、API Key 录入
- Tab 3 **行为**：启用、自动提示、划词、默认源/目标语言

### 4.4 划词面板（Selection Panel）

- 浮层定位在选区下方（空间不足时上方）
- 结构：品牌区 + 模型选择 + 词典区 + AI 上下文区 + 工具栏

## 5. 性能与体验需求

| 指标             | 目标                                                              |
| ---------------- | ----------------------------------------------------------------- |
| 首屏可译时间     | 用户点击翻译后，视口内首批结果可见 <5s（依赖网络与模型 RT）       |
| 单批大小         | ≤15 段文本，总字符约 ≤3500（含 JSON 开销）                        |
| SSE 首字节时间   | <1.5s（与上游模型有关）                                           |
| 滚动响应         | 防抖 120ms，避免频繁请求                                          |
| DOM 变更         | Mutation 防抖 300ms                                               |
| 后端 P99 延迟    | 鉴权类接口 <200ms；翻译类受上游 LLM 影响                          |
| 扩展产物体积     | gzip 后 <300KB（不含 Antd 主题 chunk）                            |

## 7. 约束与依赖

- **外部依赖**：选定 LLM 提供商可用性与配额、免费翻译 SDK 的限速
- **扩展权限**：`storage`、`activeTab`、`scripting`、`<all_urls>` 与后端 host 权限
- **后端依赖**：PostgreSQL（用户、会员、配额、SSE 会话），Redis（可选，限流）
- **密钥配置**：高级模型 Key 由后端 `.env` 持有；用户自带 Key 仅在浏览器本地

## 8. 术语表

| 术语           | 定义                                                              |
| -------------- | ----------------------------------------------------------------- |
| 逐步翻译       | 按视口区域分批翻译，滚动触发后续批次                              |
| 内容脚本       | 注入网页的 `content.ts` / `selection-translate.ts`                |
| Service Worker | 扩展后台 `service-worker.ts`，负责消息路由与 SSE 转发             |
| 跳过容器       | `pre/code` 等标签及其子树，不参与整页翻译                         |
| Tier           | 会员等级：`free` / `basic` / `premium`                            |
| Key Source     | 模型的 Key 来源：`system`（后端）/ `user`（自带）/ `none`（免费）|
