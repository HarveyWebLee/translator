# 英文网页翻译助手 — 需求功能文档

> 文档版本：v1.1.2  
> 产品形态：Chrome / Edge 浏览器扩展（Manifest V3）  
> 目标用户：需要阅读英文网页、希望获得高质量中文译文的用户

---

## 1. 产品概述

### 1.1 产品定位

**英文网页翻译助手**是一款基于 **DeepSeek 大语言模型** 的浏览器扩展。当用户浏览英文网站时，扩展可自动识别页面语言，在用户确认后，将页面正文逐步翻译为**简体中文**，并可选提供**划词词典式翻译**能力。

### 1.2 核心价值

| 价值点 | 说明 |
|--------|------|
| 智能识别 | 无需用户手动指定语言，结合 `html[lang]` 与正文采样判断是否为英文页 |
| 用户可控 | 翻译前需用户确认，避免误译与不必要的 API 消耗 |
| 高质量译文 | 使用 DeepSeek Chat Completions，支持多模型切换 |
| 按需加载 | 视口内优先翻译，滚动后逐步翻译，降低首屏等待与单次请求压力 |
| 学习辅助 | 划词模式提供音标、词性、例句与上下文释义 |

### 1.3 非目标（当前版本不做）

- 不替代专业 CAT / 本地化平台
- 不支持离线翻译
- 不保证 iframe、Shadow DOM、Canvas 内文本的完整覆盖
- 不提供持久化「双语对照」存档（刷新页面即恢复原文）

## 2. 用户角色与场景

### 2.1 用户角色

- **普通阅读者**：浏览英文新闻、博客、文档，需要整页或局部中文
- **开发者 / 学习者**：阅读英文技术文档，需要划词查义与上下文解释
- **扩展管理员**：配置 API Key、模型与功能开关

### 2.2 典型使用场景

| 场景 ID | 描述 | 期望结果 |
|---------|------|----------|
| S1 | 用户打开 BBC、GitHub README 等英文页 | 顶部出现提示条，显示检测理由与置信度 |
| S2 | 用户点击「翻译为中文」 | 视口内英文先被翻译，滚动后继续翻译下方内容 |
| S3 | 用户点击「暂不翻译」 | 当前会话不再自动弹出提示条 |
| S4 | 用户从扩展图标手动触发翻译 | 显示提示条并开始翻译流程 |
| S5 | 用户在选项页配置 API Key | 密钥保存至 `chrome.storage.sync`，扩展可正常调用 API |
| S6 | 用户选中页面英文单词/短语 | 选区附近弹出词典面板（需开启划词翻译） |
| S7 | 用户阅读含代码块的文档 | `<pre>` / `<code>` 内代码保持英文原文，不被翻译 |

## 3. 功能需求清单

### 3.1 语言检测（F-DETECT）

| 编号 | 需求 | 优先级 | 验收标准 |
|------|------|--------|----------|
| F-DETECT-01 | 读取 `html[lang]`、`meta content-language`、`og:locale` | P0 | 声明为 zh/ja/ko 时判定为非英文；声明为 en 时判定为英文 |
| F-DETECT-02 | 对可见正文采样（排除 script/style/pre/code 等） | P0 | 采样长度上限约 8000 字符 |
| F-DETECT-03 | 基于拉丁字母 / CJK 占比启发式判断 | P0 | 拉丁占比 ≥55% 且中文占比 <12% 时判定为英文 |
| F-DETECT-04 | 正文过少时降低置信度，不强行提示 | P1 | 有效字符 <40 或总采样 <80 时不自动弹条 |
| F-DETECT-05 | 页面加载后延迟检测（800ms / 2500ms 两次） | P1 | 适配 SPA 延迟渲染 |
| F-DETECT-06 | Popup 内展示当前标签页语言状态 | P2 | 显示疑似英文/非英文及 lang 属性 |

### 3.2 整页翻译（F-PAGE）

| 编号 | 需求 | 优先级 | 验收标准 |
|------|------|--------|----------|
| F-PAGE-01 | 检测到英文后顶部显示提示横幅 | P0 | 含「翻译为中文」「暂不翻译」按钮 |
| F-PAGE-02 | 用户确认后才开始翻译 | P0 | 未点击前不调用 API |
| F-PAGE-03 | 收集含英文字母的文本节点 | P0 | 单段长度 ≥2，且含 `[a-zA-Z]` |
| F-PAGE-04 | 跳过不应翻译的区域 | P0 | `pre/code/script/style` 等及其**所有子节点**（含 hljs 嵌套 span） |
| F-PAGE-05 | 视口内 + 下方 20% 预加载区优先翻译 | P0 | 使用 IntersectionObserver + 滚动防抖 |
| F-PAGE-06 | 批量调用 DeepSeek，每批最多 15 段 | P0 | 进度条显示「已译 x / y 段」 |
| F-PAGE-07 | DOM 替换时保留原文 | P1 | `data-ds-original` 存原文，`data-ds-translated` 标记已译 |
| F-PAGE-08 | 支持 SPA / 动态内容 | P1 | MutationObserver 刷新待译注册表 |
| F-PAGE-09 | 用户可停止逐步翻译 | P2 | 点击「停止逐步翻译」后不再发起新批次 |
| F-PAGE-10 | 刷新页面恢复原文 | P1 | 不持久化修改，依赖浏览器刷新 |

### 3.3 划词翻译（F-SELECT）

| 编号 | 需求 | 优先级 | 验收标准 |
|------|------|--------|----------|
| F-SELECT-01 | 选项页可开关划词翻译 | P0 | 默认关闭，需用户主动开启 |
| F-SELECT-02 | 鼠标选中英文后延迟 180ms 弹出面板 | P0 | 避免与双击选词冲突 |
| F-SELECT-03 | 返回词典式 JSON：音标、词性、释义、例句、上下文解释 | P0 | 面板分区展示词典区与 AI 解释区 |
| F-SELECT-04 | 面板内可切换模型并重新请求 | P1 | 切换后按当前选区重译 |
| F-SELECT-05 | 支持固定面板、复制、TTS 朗读 | P2 | 固定后滚动不关闭；Esc 关闭 |
| F-SELECT-06 | 选区过长时截断至 400 字符 | P1 | 防止超出模型上下文 |

### 3.4 配置与管理（F-CONFIG）

| 编号 | 需求 | 优先级 | 验收标准 |
|------|------|--------|----------|
| F-CONFIG-01 | 选项页配置 DeepSeek API Key | P0 | 必填，密码框展示 |
| F-CONFIG-02 | 可选模型：deepseek-chat / v4-flash / v4-pro | P0 | 保存至 storage |
| F-CONFIG-03 | 总开关：启用/禁用扩展 | P0 | 禁用后 API 请求拒绝 |
| F-CONFIG-04 | 自动提示开关 | P0 | 关闭后不自动弹横幅 |
| F-CONFIG-05 | 支持 `config.local.json` 自动导入 Key | P1 | 仅当 storage 为空时导入 |
| F-CONFIG-06 | Popup 快捷开关与「翻译当前页」 | P1 | 脚本未注入时自动 executeScript |

### 3.5 安全与合规（F-SEC）

| 编号 | 需求 | 优先级 | 验收标准 |
|------|------|--------|----------|
| F-SEC-01 | API Key 仅存本地（storage / 本地 json） | P0 | 不上传第三方（除 DeepSeek API） |
| F-SEC-02 | `config.local.json`、`.env` 加入 .gitignore | P0 | 防止密钥误提交 |
| F-SEC-03 | API 调用在 Service Worker 中发起 | P0 | 避免内容脚本 CORS 问题 |
| F-SEC-04 | 用户明确知情：翻译产生 API 费用 | P1 | README / 选项页说明 |

## 4. 交互与界面需求

### 4.1 顶部提示横幅（Content Script）

- 固定于页面顶部，不遮挡主要导航（z-index 高于正文）
- 状态：待翻译 → 逐步翻译中（进度）→ 翻译完成 / 已停止
- 加载态：主按钮禁用并显示「正在连接 DeepSeek…」

### 4.2 弹出窗口（Popup）

- 显示当前页语言检测状态圆点（绿/灰）
- 开关：启用扩展、自动提示、划词翻译
- 按钮：翻译当前页、重新检测
- 链接：打开选项页

### 4.3 选项页（Options）

- 表单：API Key、模型、三个行为开关
- 保存后即时提示「已保存」

### 4.4 划词面板（Selection Panel）

- 浮层定位在选区下方（空间不足时上方）
- 结构：品牌区 + 模型选择 + 词典区 + AI 上下文区 + 工具栏

## 5. 性能与体验需求

| 指标 | 目标 |
|------|------|
| 首屏可译时间 | 用户点击翻译后，视口内首批结果 <5s（依赖网络与 API） |
| 单批大小 | ≤15 段文本，总字符约 ≤3500（含 JSON 开销） |
| 滚动响应 | 防抖 120ms，避免频繁请求 |
| DOM 变更 | Mutation 防抖 300ms |
| 内存 | 不长期持有已断开节点的引用（注册表过滤 `isConnected`） |

## 7. 约束与依赖

- **外部依赖**：DeepSeek API 可用性与配额
- **权限**：`storage`、`activeTab`、`scripting`、`<all_urls>`、`https://api.deepseek.com/*`
- **密钥配置**：`.env` 仅作开发参考，**扩展运行时不会读取 `.env`**

## 8. 术语表

| 术语 | 定义 |
|------|------|
| 逐步翻译 | 按视口区域分批翻译，滚动触发后续批次 |
| 内容脚本 | 注入网页的 `content.js` / `selection-translate.js` |
| Service Worker | 扩展后台 `service-worker.js`，负责 API 调用 |
| 跳过容器 | `pre/code` 等标签及其子树，不参与整页翻译 |
